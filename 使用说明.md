# 多模态数据标注辅助工具使用说明

## 1. 启动应用

项目使用 Anaconda `python3.10` 环境。在项目目录打开 PowerShell：

```powershell
conda run -n python3.10 python -m pip install -r requirements.txt
conda run -n python3.10 streamlit run app.py --server.port 8501
```

浏览器访问 `http://localhost:8501`。

## 2. 快速体验演示数据

项目提供 RGB、Depth、FSR、IMU 和超声波演示数据生成器。生成后的文件位于 `data/`，该目录不会提交到 Git 仓库。

1. 打开“数据”页面并点击“生成演示数据”。
2. 生成完成后点击“重新加载”。
3. 确认四个设备均显示为已加载。
4. 打开“对齐与分割”，点击“对齐数据”。
5. 打开“标注”开始区间标注。

也可以运行 `conda run -n python3.10 python generate_sample_data.py` 生成相同数据。

## 3. 项目与设备配置

### 导入 Schema

在侧边栏“项目配置”上传 YAML 文件。Schema 用于描述：

- 主设备和其他传感器设备；
- CSV、图像及 Depth 数据路径；
- 时间戳列、数值列和图像文件名列；
- 采样频率与插值策略；
- 标签、边界过渡、变点检测及模型配置。

默认示例位于 `config/sensors_schema.yaml`。

### 管理设备

在侧边栏打开“管理设备”，可以新增、修改或删除设备。修改设备名称、类型、路径、时间戳列、数值列或插值方式后，需要重新加载并对齐数据。

### 选择主轨

“主设备”的时间戳将作为统一帧网格。通常选择 RGB-D 相机等能够代表最终样本帧率的设备。

## 4. 自定义标签组与标签

在侧边栏点击“管理标签”，或在“标注”页面点击标签下拉框旁的编辑按钮。

### 标签组

“标签组”用于定义同一类标签之间的选择规则：

- 单选组：同一帧在该组中只能有一个标签，适合地形、动作、步态阶段等互斥类别；
- 多选组：同一帧在该组中可以同时有多个标签，适合事件、属性、质量问题等可共存类别。

在“标签组”页签中可以新增、改名和切换选择方式。组内仍有标签或区间时不能删除。已有多标签区间的组不能直接切换为单选组。

### 新增标签

1. 打开“标签”页签，在底部输入新标签名称。
2. 选择该标签所属的标签组。
3. 使用颜色选择器设置时间轴显示颜色。
4. 点击“添加”。

### 修改标签

1. 修改已有标签的名称、颜色或所属组。
2. 点击该行的保存按钮。

标签重命名后，已有标注区间、撤销记录和当前模型建议会同步更新。修改颜色后，时间轴中的已有区间也会立即使用新颜色。

正在被区间使用的标签可以改名或改色，但不能直接移动到其他组。需要先删除相关区间，再修改所属组，避免多标签区间被错误拆分。

### 删除标签

未被任何区间使用的标签可以直接删除。正在使用的标签会禁用删除按钮；需要先删除对应标注区间，或者将标签重命名。

标签配置属于当前 Schema，导出 Schema YAML 后可以在其他项目中复用。

旧版 Schema 若没有 `label_groups`，加载时会自动创建名为“类别”的单选组，并把原有标签放入该组，无需手动迁移。

## 5. 加载数据

“数据”页面按照 Schema 列出所有设备。每个设备可以使用配置路径，也可以临时上传文件。

### 数值设备

CSV 至少需要包含：

- 配置的时间戳列；
- Schema 中声明的数值列。

时间戳支持 Unix 秒、毫秒、微秒、纳秒以及 ISO 8601 字符串。

### 图像设备

图像设备需要时间戳 CSV 和图像目录。通过界面上传时可使用 ZIP，ZIP 中应包含 PNG/JPG 图像，并可包含一个时间戳 CSV。

## 6. 数据对齐与检查

### 主轨对齐

1. 设置“对齐容差”。
2. 点击“对齐数据”。
3. 查看对齐帧数、数据列、缺失值和重复时间戳。
4. 在“数据质量明细”检查各列缺失率。

### 冲击峰值校正

存在踩地、敲击等同步动作时，可以选择 FSR 或 IMU 信号执行“校正时间偏移”。系统检测冲击峰值、修正设备时间偏移并重新对齐。

### 逐帧对齐检查

对齐完成后，页面底部显示剪辑软件式帧检查器：

- 使用滑块定位任意帧；
- 使用前一帧、后一帧按钮逐帧移动；
- 同时查看当前 RGB、Depth 和传感器数值；
- 使用七帧缩略带检查前后视觉连续性。

重点观察动作发生的图像帧是否与 FSR、IMU 等信号变化同时出现。若不同步，可调整容差或执行冲击峰值校正。

## 7. 自动分割

在“对齐与分割”中选择变点算法、代价模型和检测信号，然后点击“检测变点”。检测结果会显示在标注时间轴上，作为区间边界参考。

常用参数：

- 惩罚项越大，检测到的变点越少；
- 最短区间样本数用于过滤过短片段；
- 检测步长越大速度越快，但边界精度会降低。

## 8. 区间标注

标注页顶部为固定工具栏，包含当前标签组、标签、应用、下一空白、撤销和标签管理。滚动查看预览或长时间轴时，主要操作仍保持可见。

1. 在多轨时间轴上拖选时间范围，或移动播放指针后使用 `I` 设置入点、`O` 设置出点。
2. 选择标签组。
3. 单选组选择一个标签；多选组可选择一个或多个标签。
4. 点击顶部工具栏中的“应用”。
5. 使用“下一空白”跳转到当前标签组的下一段连续空白区间。

标注进度、已标注帧、未标注帧、Ignore 帧和标签分布均按当前标签组单独统计。

### 一帧对应多个标签

不同标签组的结果可以同时存在于同一帧。例如某帧可以同时是：

```text
地形：楼梯
动作：上楼
步态阶段：支撑期
事件：打滑、遮挡
```

同组内的处理规则如下：

- 单选组的区间发生重叠时，后创建的区间在重叠帧上生效，不会影响其他组；
- 多选组的区间发生重叠时，重叠帧取标签并集，例如“打滑”和“遮挡”得到 `打滑|遮挡`；
- 删除某个多选区间后，系统会根据剩余区间重新计算逐帧标签；
- 多选组留空表示尚未标注。若要表达“已经检查且没有事件”，建议在该组新增“无事件”标签。

### 修改标注结果

- “撤销上次”删除最近应用的标签并恢复原选区；
- “清空标注”删除全部区间；
- 已标注区间列表支持跳转和单独删除；
- 起止点可以落在同一时间戳上，用于补标单独一帧；
- 修改标签名称或颜色会同步已有区间。

### 质量检查

时间轴下方的“质量检查”会生成可跳转的复核队列：

- 未标注片段：当前标签组中仍为空的连续帧；
- 过短区间：帧数低于自定义阈值的区间，可能是误选或边界抖动；
- 单选组重叠：同一单选组中不同标签的区间发生重叠，后创建的标签虽然会生效，但建议人工确认。

可按标签组筛选问题，点击“跳转”会把播放指针和选区移动到问题位置。多选组的标签并集是合法行为，不会被当作冲突。

### 保存和恢复草稿

“草稿与恢复”用于避免浏览器刷新或会话中断造成返工：

1. 点击“下载标注草稿”保存 JSON 文件；
2. 恢复时上传该 JSON，选择“替换当前标注”或“合并到当前标注”；
3. 点击“恢复草稿”。

恢复操作会先完整校验标签组、标签和时间范围。草稿有任意一个区间无效时，当前标注不会被部分覆盖。Master CSV 导出时生成的 `_intervals.json` 也可以作为恢复来源。

清空全部标注已移到该辅助区，并要求先勾选确认，避免主操作区误触。

### 边界过渡区

在“设置 > 标注”中启用 Ignore 过渡区并设置过渡半径。相邻且标签不同的连续区间将在交界处自动标记为 Ignore。

## 9. 模型辅助标注

在侧边栏点击“配置模型 API”，输入：

- 模型名称；
- OpenAI 兼容 API 地址；
- API Key。

选择标签组和区间后点击“分析选区”，模型会根据传感器统计特征和可用图像给出建议。确认结果后点击“采纳建议”。模型只能返回当前标签组中的标签，采纳结果也只写入该组。多选组目前每次建议一个标签，需要多个标签时可人工多选后应用。

API Key 只保存在当前运行会话中，不会写入导出的 YAML 或 HDF5。

## 10. 导出数据集

进入“导出”页面检查各标签组覆盖率和数据预览，然后选择格式：

- HDF5：包含对齐数据、区间、分组标签编码和 Schema；
- Master CSV：包含逐帧对齐结果和每个标签组的列，并附带区间 JSON；
- Schema YAML：保存设备、标签及处理配置。

CSV 中每组使用独立列，例如 `label__地形`、`label__动作`、`label__步态阶段` 和 `label__事件`。多选值用 `|` 分隔。兼容列 `label` 保留默认组（第一个标签组）的结果。

HDF5 中单选组导出为 `label_id__<组名>`，空值为 `-1`，Ignore 为 `-2`；多选组导出为 `label_multihot__<组名>` 多热矩阵，其 `label_mapping` 属性记录标签与列索引的对应关系。标签字符串列也会保留，便于检查和跨工具读取。

存在未标注帧时仍可导出，对应组的标签保持为空。API Key 不会写入 HDF5 或 Schema YAML。

## 11. 常见问题

### 标签下拉框为空

打开“管理标签”并至少新增一个标签。

### 图像与传感器变化不同步

检查时间戳单位、主设备选择和对齐容差；存在同步冲击动作时使用冲击峰值校正。

### 对齐后出现大量缺失值

适当增大对齐容差，确认各设备时间范围存在重叠，并检查时间戳解析是否正确。

### 无法删除标签

该标签仍被标注区间使用。先在已标注区间列表删除相关区间，或直接重命名该标签。

### 多选组中的空值是否表示“无事件”

不是。空值默认表示该帧尚未完成该组的标注。若训练或质检需要明确区分，新增“无事件”标签并对确认无事件的范围进行标注。
